Basic Labels

Published May 26, 2022

The basic labels such as h1, h2, h3, and others, let us to keep a clean distinction around the set of topics we want to explain in an article.

  • h1 label for HEADINGS of level 1 for TITLES

  • h2 label for HEADINGS of level 2 for SUBTITLES

  • h3 label for HEADINGS of level 3 for SUBTITLES

  • h4 label for HEADINGS of level 4 for SUBTITLES

  • h5 label for HEADINGS of level 5 for SUBTITLES

  • h6 label for HEADINGS of level 6 for SUBTITLES

< !DOCTYPE > Label

It's use to specify what type of html doc the sever it's gonna read. Usually it's written 'html' after !DOCTYPE due to the ! which mean that this label own an atribute

The Webpage Hierarchy

Every webpage it's divided by two parts.

HEAD. - Which it's conformed by the TITLE of the wpage and the URL which contains foundamental settings for the entire wpage.

HTML. - This is the content of the entire page such as body, pictures, vids, hyperlinks and other stuff. So, For that reason of hierarchy, it's highly recommended to follow the *html labels* and structure:

HTML STRUCTURE

  • < !DOCTYPE html >
  • < html >
  • < head >
  • // label for head exa: title
  • < /head >
  • < body >
  • // label for body exa: h1, p
  • < /body >
  • < /html >

Attributes

Published May 26, 2022

< html lang="en" >

This attribute it's use to indicate what language belongs to the wpage. This has many advantages like to be easily found in a country depending on its *language*. Also... this attribute it's used in the html label. exa:

  • html lang="en"
  • lang = language
  • "en" = english (it could be spanish)

< meta / >

  • This label doesn't have something inside it because this doesn't have another pair to *close* the instruction.
  • Normally it's used to *approach* its *_attributes_* which *let to give more information* about our web page.
  • Meta label it's not neccessary to close with a / before the >.

Some of the used attributes:

----- name="description / autor / keywords" content="(info to show)",

----- charset="",

----- and others...
